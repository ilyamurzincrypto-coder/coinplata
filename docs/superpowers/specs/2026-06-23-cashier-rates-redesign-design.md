# Дизайн: переделка блока «Курсы» на главной Кассира

**Дата:** 2026-06-23
**Статус:** утверждён дизайн (юзер: «погнали»)

## Проблема
Текущий блок курсов на главной (`RatesSidebar`) — «портянка»: непонятный набор пар
в две колонки, не совпадает с привычной менеджерам Google-таблицей. Менеджеры
ориентируются на свой лист (направленные пары `USDT↔USD/TRY/EUR`, USD в %, остальное
абсолютом) и хотят такой же вид — компактно, красиво, понятно — плюс быстрый ввод
курсов «пастом», как в проекте coinpoint.

## Цель
`RatesSidebar` = две секции:
1. **Мастер-курсы (USDT кеш-кеш)** — ровно их 6 строк, редактируемые (паст + инлайн).
2. **Автокурсы (кеш-кеш)** — производные кросс-курсы, read-only, ниже.

## Текущее состояние (что переиспользуем)
- UI: `src/components/RatesSidebar.jsx` (+ `components/rates/RatesTable.jsx`, `RateCell`).
- Стор: `src/store/rates.jsx` — модель Валюта→Канал→Пара; API `getRate`, `setRate(from,to,value)`,
  `addPair`, `confirmRates/unconfirmRates`, `getOfficeOverride`, `setSpecialRatesSnapshot`.
- Данные: Supabase `public.pairs` (default pairs) + `public.office_rate_overrides`.
- Импорт: `RatesImportModal` (xlsx + текст), `utils/morningRatesParser.js`, RPC
  `rpcImportRates`, `rpcUpsertOfficeRate`. Лайфцикл курсов `draft→confirmed` + баннер.

## Дизайн

### Секция 1 — Мастер-курсы (USDT кеш-кеш)
Фиксированные 6 строк (направленные):
```
USDT → USD   −1,00 %      USD↔USDT  → формат %
USD  → USDT  +0,20 %
USDT → TRY    45,10       TRY/EUR   → абсолют
TRY  → USDT   46,00
USDT → EUR     1,177
EUR  → USDT    1,16
```
- Источник значения — эффективный курс пары (override офиса поверх default pair).
- **Формат отображения/ввода:** для пар USD↔USDT — проценты `% = (rate − 1) × 100`
  (т.е. `rate = 1 + %/100`); для TRY/EUR — абсолютный `rate`. Признак «percent-режима»
  задаётся списком «парных» валют (`PERCENT_CCYS = ["USD"]`, расширяемо).
- **Inline-правка:** клик по значению → инпут в том же формате → парс → `setRate(from,to,rate)`
  (для %-режима из введённого % считаем `rate`). Enter/blur — применить, Esc — отмена.
- **Паст:** кнопка «Вставить курсы» открывает textarea; вставляешь блок их формата
  (`USDT -> USD  -1,00%` / `USDT -> TRY 45,10`); парсер → diff-превью (было→стало,
  new/updated/unchanged/ошибка) → «Применить» пишет через `setRate`/`rpcImportRates`.
- Лайфцикл `draft→confirmed` сохраняется: правки → draft, баннер «подтвердить курсы».

### Секция 2 — Автокурсы (кеш-кеш), read-only
- Кросс-курсы между `USD/EUR/TRY/RUB` (нал) в обе стороны, выведенные из мастера через
  USDT-пивот (`convert(from,to)` триангуляцией). Примеры: `USD→TRY`, `TRY→USD`,
  `EUR→TRY`, `USD→EUR`, `RUB→TRY`, …
- Считаются на лету из эффективных мастер-курсов, **не редактируются**, помечены «авто»
  (приглушённый цвет + значок). Меняются автоматически при правке мастера.

### Офисы
Компактный свитчер (Global + активные офисы) остаётся. Global = default pairs;
выбран офис → эффективные с учётом `office_rate_overrides`. По умолчанию — Global.
(Если мастер строго глобальный — табы убираются тривиально.)

### Парсер пасты — новый модуль `utils/ratesPasteParser.js`
- Вход: многострочный текст. На строку: `<FROM> <arrow> <TO> <value>`.
- Нормализация: стрелки (`->`, `→`, `>`), десятичная запятая→точка, `%` опционально,
  пробелы/регистр валют.
- Маппинг на пары; для %-валют `rate = 1 + %/100`, иначе `rate = value`.
- Выход: `[{from,to,rate,raw,status}]` (`new|updated|unchanged|error`) для превью.
- Покрыто unit-тестами (vitest) — happy path, запятая, %, стрелки, мусорные строки.

### Стиль
Компактная таблица в духе их листа: моноширинные числа, выравнивание вправо, зебра,
тонкие разделители, зелёный/красный для ±%. Заголовки секций «Мастер» / «Авто».
В стиле приложения (Tailwind токены проекта). Адаптив сохраняем.

## Решения
- **% — только формат**, в БД абсолютный `rate`. Без новой схемы/миграций.
- **Без новых таблиц**: пишем в существующие `pairs`/`office_rate_overrides` через
  существующие `setRate`/`addPair`/`rpcImportRates`/`rpcUpsertOfficeRate`.
- **Автокурсы — вычисляемые**, не хранятся.

## Вне scope (позже)
- Публичный API курсов для сайта (контракт согласован отдельно: каналы, office→city, снапшот).
- Разделение курсов по каналам bank/sepa/swift (сейчас один курс на пару).

## Тестирование
- `ratesPasteParser` — unit (vitest).
- Конвертация %↔rate — unit (граничные: 0%, −1%, +0,2%).
- Автокурс-вывод (USDT-пивот) — unit на паре примеров.
- Ручная проверка в dev (seed-режим): inline-правка, паст-превью+apply, переключение офисов,
  секция авто пересчитывается.

## Acceptance
- Секция 1 = 6 строк их формата; USD в %, TRY/EUR абсолютом; inline + паст работают, пишут в БД.
- Секция 2 показывает нал-кросс `USD/EUR/TRY/RUB`, read-only, пересчёт при правке мастера.
- Лайфцикл draft→confirmed и баннер не сломаны. Сборка `npm run build` зелёная, тесты проходят.
